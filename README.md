# Amsterdam Tigers

Rebuilt website for the Amsterdam Tigers ice hockey club. Next.js 14 (App
Router, TypeScript), Tailwind, Framer Motion and Lenis. Evolves the club's
black / red / white identity with a crest-assembly loader, a video hero, a 3D
scroll-tilt team lineup and scroll-reveal motion throughout.

## Run it

```bash
npm install
npm run dev
```

Open http://localhost:3000. Production build:

```bash
npm run build
npm start
```

> The first build downloads the Archivo and JetBrains Mono fonts via
> `next/font/google`, so the machine needs internet access on that first run.

## Stack

- Next.js 14 App Router + TypeScript
- Tailwind CSS with OKLCH design tokens (see `src/app/globals.css`)
- Framer Motion (loader, hero parallax, 3D card tilt, reveals, custom cursor)
- Lenis smooth scroll (auto-disabled under `prefers-reduced-motion`)
- `PixelCard` (adapted from React Bits, MIT) on the social feed tiles
- Custom cursor (dot + trailing ring, desktop fine-pointer only)

## Structure

```
src/
  app/            routes (home + team + one page per nav item)
  components/     Nav, Hero, Loader, TeamLineup, PlayerCard, SocialFeed, ...
  lib/            site config, roster data, crest geometry, motion curves
public/
  favicon.*, icon-*.png, site.webmanifest   generated crest icons
  media/          drop hero.mp4 + player photos here (see media/README.md)
```

## Assets

The real club assets are wired in:

- **Hero**: `public/media/hero.mp4` (muted 24s loop, re-encoded from the source
  55 MB clip down to ~2.3 MB) + `hero-poster.jpg`.
- **Crest / favicons**: generated from the club logo, recolored white for the
  dark UI (`public/media/crest-white.png`); hero uses the full lockup
  (`logo-lockup.svg`).
- **Players**: 24 portraits in `public/media/players/`, mapped in
  `src/lib/players.ts`.
- **Partners**: logos in `public/media/partners/`, normalized to white.
- **History**: the 1934 Linnaeusstraat rink photo on `/history`.
- **Feed**: `SocialFeed` uses real club imagery as a fallback grid.

## Deploy

Works out of the box on Vercel: push to GitHub, import the repo, deploy. No env
vars required for the base site.

## Push to GitHub

The repo is already initialised with an initial commit. To publish:

```bash
# with the GitHub CLI
gh repo create amsterdam-tigers --private --source=. --remote=origin --push

# or manually, after creating an empty repo on github.com
git remote add origin git@github.com:<your-username>/amsterdam-tigers.git
git branch -M main
git push -u origin main
```

## SEO

- **Metadata** (`src/app/layout.tsx`): title template, Dutch description,
  Open Graph (`nl_NL`), Twitter `summary_large_image`, and `robots` directives.
- **`metadataBase` is environment-driven** (`src/lib/site.ts` -> `SITE_URL`):
  `NEXT_PUBLIC_SITE_URL` -> Vercel production URL -> `localhost`. Set
  `NEXT_PUBLIC_SITE_URL` in Vercel once a custom domain is live so canonicals,
  Open Graph URLs, sitemap and JSON-LD all point at the real domain.
- **Share image**: `src/app/opengraph-image.png` + `twitter-image.png`
  (1200x630, generated from the lockup) via Next's file convention; applies to
  every route with correct dimensions and alt text.
- **`sitemap.xml`** and **`robots.txt`** are generated by
  `src/app/sitemap.ts` / `src/app/robots.ts`.
- **Structured data**: `SportsTeam` + `WebSite` JSON-LD in the layout. Verify
  the venue/address (currently Jaap Edenbaan, Amsterdam) is correct.
- **Canonicals** are set per route; one `<h1>` per page.

Post-deploy: submit the sitemap in Google Search Console and confirm the
JSON-LD in the Rich Results Test.
